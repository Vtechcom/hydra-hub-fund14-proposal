## 3. Hydra Operations Test Procedure

Each developer performed the following operations through the Hydra Hub Consumer Portal:

1. **Head Open** — Initialize a Hydra Head via the Consumer Portal
2. **UTXO Commit** — Commit UTxOs into the Hydra Head
3. **Transaction** — Submit transactions within the Hydra Head
4. **Fanout** — Execute fanout transaction, returning funds to Cardano L1

In addition to screenshots, the Hydra node state log ([`state`](../state)) provides machine-readable proof of all developer operations. The following lifecycle events were recorded:

| Event | Count | Verification |
|-------|-------|-------------|
| `HeadInitialized` | 1 | Head Open confirmed on-chain |
| `CommittedUTxO` | 2 | UTXO commits recorded |
| `HeadOpened` | 1 | Head successfully opened |
| `TransactionReceived` | 1 | Transaction submitted in Head |
| `TransactionAppliedToLocalUTxO` | 1 | Transaction applied to local state |
| `SnapshotRequested` | 1 | Snapshot requested after transaction |
| `PartySignedSnapshot` | 3 | Parties signed snapshots |
| `SnapshotConfirmed` | 1 | Snapshot confirmed |
| `HeadClosed` | 1 | Head closed on-chain |
| `HeadIsReadyToFanout` | 1 | Fanout ready |
| `HeadFannedOut` | 1 | Fanout completed, funds returned to L1 |

This log confirms the complete Hydra lifecycle: **Head Open → UTXO Commit → Transaction → Snapshot → Head Close → Fanout**.
